

The importer is started with

bin/cm my-importer start

In a Windows environment the importer is installed as a service and can optionally be launched from the corresponding menu item or from the Windows Services window. 

According to the configuration, the importer runs permanently or terminates itself. When configured to run permanently (Unix only) it can be stopped with

bin/cm my-importer stop

In a Windows environment you stop the importer from the corresponding menu item or from the Windows Services window.

Furthermore, the importer can be restarted (Unix only) when executing

bin/cm my-importer restart

Initialisation of components is described in general in Section 5.1.3.1.
