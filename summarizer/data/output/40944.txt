

But nevertheless, these basic principles go with any CoreMedia workflow:

The user accepting a task first becomes the preferred user of this group. Within a workflow tasks of this type will always be suggested to this user first. If the preferred user rejects a task, it is offered to all other members of the group.
Rejected tasks are automatically sent back to their last editor.
Not all users of a group can reject a task: the last user has to accept it.
