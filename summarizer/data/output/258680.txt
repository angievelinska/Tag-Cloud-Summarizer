

In order to define the bean type model, you must define bean types and register them at the bean type repository.

Currently, a few restrictions apply to the bean type definitions:

Bean types and properties need to be explicitly defined for the Content Rules framework, no mechanism is provided to derive the model structure from the CAE contentbeans.xml definitions or java bean classes.
Only few property types are supported, namely the ones that can be handled by the single bean repository type supported.

Bean types are defined the file WEB-INF/spring/editor-extensions/beanqueryeditor-beantypes.xml using the Spring Framework bean configuration scheme.
